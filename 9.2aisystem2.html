智能客服 & AIoT 综合测试

知识点总览

本套题共 70 题，围绕 智能客服系统、AIoT 终端与人机交互、云-端系统设计 以及 知识库管理 (KBMS) 四大板块，重点考查：

  • 数据整合 & 预处理：数据清理、变换、标准化及多模态融合。
  • 前端 UI / UX：聊天机器人组件、可访问性三原则、跨设备适配。
  • NLP & 知识库：语义检索、FAQ 匹配、情感分析、Transformer。
  • 生成模型 & 评估：温度采样、F1 指标、模型剪枝与性能权衡。
  • AIoT / 云服务：数据安全、实时告警、REST API、分布式计算。
  • KBMS：RDF/OWL 表示、知识质量三维度、语义检索。

请在答题过程中关注 核心概念区分 与 跨领域共性（如“先数据→模型→UI”）。

得分：0 / 70

1. 智能客服系统整合异构数据最常用的 AI 技术是（ ）

机器学习通过特征工程将文本、数值、日志等转换为统一向量，使多源数据能在同一空间被分析；NLP/视觉仅处理单一模态，数据挖掘更偏后期统计。

2. 整合客户数据的首要步骤是（ ）

“扫地”在先——去重、补缺、统一时间格式，否则后续统计分析将被脏数据拖偏。

3. 把社交媒体、CRM、日志格式对齐的过程叫（ ）

数据变换强调字段重命名、编码重排，属于“结构映射”；标准化 / 归一化仅针对数值缩放。

4. 哪类数据最难整合进客服系统？（ ）

视频 = 帧序列 + 声音 + 字幕，体积最大、模态最多，存取、标注、转码都最费时。

5. 下面哪个行业对客服数据整合需求最高（ ）

金融行业需合并多渠道历史数据以支撑 KYC、反欺诈和精准风控。

6. 与客户直接对话的组件是（ ）

Chatbot 负责收发消息，是“面孔+嘴巴”；NLP/ASR/知识图谱位于后台提供能力。

7. 网页客服 UI 常用前端技术栈（ ）

AJAX 负责异步请求，HTML5 构建结构，CSS3 美化界面——三件套缺一不可。

8. 让用户感觉“回答不死板”的关键 UI 元素是（ ）

相同含义换多种说法，搭配表情 / gif，可显著提升亲和力与“真人感”。

9. 以下哪项是基础却必备功能（ ）

没自动回复就只是留言板；情感、转接、搜索可视业务阶段逐步升级。

10. 设计 UI 时最难攻克的是（ ）

“听懂+答对”依赖后台语义解析与知识检索，纯前端无能为力。

11. “用 AI 能让界面直接处理自然语言”——对吗？

可在浏览器跑 WebAssembly 版分词或调用云函数，前端即可做关键词检测与高亮。

12. “客服 UI 必须自带情感分析”——对吗？

情感分析通常在服务端集中完成；UI 只负责展示结果，非硬性需求。

13. 最常见的知识库结构是（ ）

以表格 / JSON 形式存放标准问答，查询速度快、易维护。

14. 自动问答的“一把钥匙”是（ ）

语义匹配、知识检索、自然语言生成均属于 NLP 范畴。

15. 客服知识库通常囊括（ ）

缺失任一类信息都会导致回答不精准。

16. 检索 FAQ 最常见的 NLP 技术是（ ）

将用户问句向量化后与 FAQ 向量库做相似度检索，效果最佳。

17. 处理“不确定问法”首选（ ）

同义词扩展、上下位词、槽位补全均属于深层语义处理。

18. “NLP 是客服的关键支柱”——对吗？

没有分词、意图识别、槽位抽取，客服只能全人工。

19. “知识库得把所有问题都列全”——对吗？

长尾问题无穷无尽；可通过生成式回答或回退人工来补充。

20. 情感分析经典算法是（ ）

SVM 对高维稀疏文本向量泛化能力强，经典且鲁棒。

21. 如今翻译质量最好的框架是（ ）

自注意力机制能处理长距离依赖 → BLEU 分最高。

22. 训练 NLP 模型第一要务是（ ）

垃圾进垃圾出——数据干净 / 标准化是高质量模型的前提。

23. 想让回答“别每次一句话”常调的参数是（ ）

采样温度越高 → 生成更随机，句式更丰富。

24. 对话模型综合评估常看（ ）

F1 综合惩罚漏答和错答，是对话评估的黄金指标。

25. “模型训练完全自动化”——对吗？

人工仍需负责标注质检、调参、早停等关键环节。

26. “数据量越大，模型表现必定变好”——对吗？

在噪声可控、模型容量够用的前提下，更多语料通常带来更广知识覆盖。

27. 想让回答覆盖面更广，最直接的做法（ ）

知识面≈模型见过的文本量；先喂“书海”才能“谈笑风生”。

28. 客服模型剪枝主要为了（ ）

瘦身可降 30‑70% 推理时延，适合移动端或高并发场景。

29. AIoT 最常见的人‑机交互通道（ ）

手机 App / 小程序底层均通过 REST/GraphQL 接口调度设备。

30. AIoT UI 首要三原则（ ）

能用 → 人人都会；可访问 → 老人/残障可用；可维护 → 后期升级不崩。

31. IoT 产品多采用的设计理念（ ）

从场景和行为出发，避免“堆功能却不好用”。

32. 处理敏感数据（如智能门锁）第一优先考虑（ ）

密码泄露比“页面卡顿”严重得多，务必先加密 & 权限控制。

33. 轻量网页控制面板常用技术（ ）

嵌入式 MCU 资源有限，原生三件套最轻量。

34. “UI 设计需考虑设备尺寸 / 物理限制”——对吗？

手环仅能滑动，冰箱屏可手写——交互逻辑需因设备而异。

35. AIoT 部署量最大的行业（ ）

“工业 4.0” 场景多、设备密度高，对 AIoT 需求最旺盛。

36. 智能家居最核心卖点（ ）

把不同品牌的灯、空调、音箱拉到一张网，体验立刻提升。

37. 智能城市应用主要集中在（ ）

城市级平台往往同时接入交通 / 环保 / 安防子系统。

38. 推进速度最慢的领域（ ）

医疗监管严格、隐私要求高，落地流程最漫长。

39. AIoT 三大支柱技术是（ ）

感知 (IoT) + 智能 (AI) + 存储/算力 (Cloud) 缺一不可。

40. “AIoT 必须同时用到 AI 和 IoT”——对吗？

无 IoT = 无源数据；无 AI = 只上传不分析。

41. AIoT 最大风险（ ）

摄像头被黑 = “在家裸奔直播”，风险最高。

42. 最易导致整个 IoT 链路瘫痪的问题（ ）

掉线 = 远程设备全部失联，业务瞬间中断。

43. 排查网络故障首选工具（ ）

抓包直观呈现 MQTT/HTTP 是否丢包、握手失败。

44. 实现实时异常告警常用（ ）

时序异常检测模型可自动适应波动，比静态阈值更精准。

45. 第三方最爱调用的云接口类型（ ）

URL + JSON 语言无关，调试友好，占据绝对主流。

46. 公网数据传输首选协议（ ）

HTTPS = HTTP+TLS，加密 + 身份校验双保险。

47. 设计云控制台首先关注（ ）

按钮排列不清→用户出错率飙升，反而拖慢业务。

48. “前端也可以用 NLP 完成多语言对话框”——对吗？

浏览器可加载 WASM 版分词、语言检测小模型，离线也能跑。

49. 采用云智能服务最多的行业（ ）

风控、量化交易、客服坐席均需弹性算力与高可用性。

50. 描述云服务必提的三特性（ ）

缺一即意味着停机、无法横向扩容或泄密。

51. 云端批处理 TB 级日志最常用（ ）

Spark/Hadoop 把任务切片并行跑，吞吐量最高。

52. 存‑算‑析依次对应云功能（ ）

数据湖 → Spark → BI/DL，从原始日志到洞察全链路。

53. 安全传输层协议是（ ）

TLS 完成密钥交换 + 对称加密；HTTPS 即 HTTP over TLS。

54. “公有云已完全替代本地机房”——对吗？

金融、军工、超低时延边缘场景仍保留本地或混合云。

55. “上云能降 CAPEX & OPEX”——对吗？

硬件改租赁，按需付费 + 运维外包，资本支出 & 运营支出均降低。

56. 云系统性能、日志、告警统管组件是（ ）

CloudWatch、Azure Monitor 等属于平台层能力。

57. 监控云健康常用的 3 类工具（ ）

对应“慢、崩、被黑”三类核心风险。

58. 提高云可靠性的一揽子做法（ ）

多活、优化、加密——可靠性、性能、安全三管齐下。

59. KBMS 的英文全称（ ）

专门用于知识库创建、维护、检索与共享的软件系统。

60. KBMS 基础功能括号里哪项描述最完整（ ）

缺“共享/应用”＝无人能用，系统价值骤减。

61. 主流 KBMS 类型（ ）

关系库存结构化条目，本体库存语义网，互补使用最普遍。

62. 机器可推理的知识表示（ ）

RDF/OWL 采用三元组 / 本体，算法可沿关系自动推理。

63. 知识管理可通过哪些手段完成（ ）

规模越大越需要“人 + 系统 + 存储阵列”协同。

64. “KBMS 目标是让知识自动流转”——对吗？

自动同步、版本控制、推送，才能让知识随处可用。

65. 知识收集要覆盖（ ）

丰富度提升 → 检索与推理场景更全面。

66. 收集阶段最大的三座山（ ）

只拼抓取量不顾质量 → 检索误导用户。

67. 评估知识质量的 3 种路径（ ）

专家给权威，指标给效率，用户给真实场景反馈。

68. 自动化 UI 测试首选（ ）

录制浏览器操作脚本，高仿真自动回归测试。

69. 最智能的知识检索方式（ ）

Embedding + 向量索引能识别同义 / 上下位，命中率最高。

70. 衡量知识价值要看（ ）

用得多 + 来源真 + 影响广 → 价值更高，优先维护。
